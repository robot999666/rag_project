# RAG模块实现逻辑

PDF → 结构化解析 → 语义增强 → 分块 → 向量库（索引）→  查询 → 改写 → 召回 → 重排 → 过滤 → 生成

```clojure
文件哈希检测
			只处理“新增 or 修改”的文件
			避免重复计算
孤儿数据清理
			如果PDF删除 → 自动删除解析结果
			数据源、向量库 一致
```

```clojure
MinerU解析pdf布局、表格、公式等

表格 → 转换成markdown标签
	把“结构信息”转成“语义信息”
```

```clojure
分块策略
第一层：Markdown结构分块
	知识点结构
	章节语义
第二层：递归切分
	控制token长度
	保证上下文连续
```

```clojure
向量模型策略embeddings
	BAAI/bge-m3 + normalize_embeddings=True
	支持多语言
	归一化 → 适配 cosine（余弦相似度）
```

```clojure
查询改写（Query Rewrite）
	指代消解（它 / 这个）
	补全上下文
```

```clojure
向量召回（Recall）
	top_k = 10
	使用 cosine
	similarity = max(0, 1 - distance)
```

```clojure
重排
	Cross-Encoder 重排
				bge-reranker-base
				(query, doc) → 相关性打分
				按logits排序
	Sigmoid归一化
				score = 1 / (1 + e^-logit)
				转成 [0,1]
	阈值过滤（防止幻觉）
				score >= 0.45
```

```clojure
生成策略（Prompt工程）
		任务分流设计
		判断：
			新题目：
					思路 → 步骤 → 答案 → 鼓励
	  	非新题目：
					解释 + 连贯对话
```

```clojure
历史裁剪
		只保留5轮
		防止：token爆炸、干扰检索
```

# 问题

1. 知识库的动态更新：便于测试与维护，增删改查功能优化

2. 索引：添加文档质量过滤（过滤空白/低质量块）

3. 数据标准化（统一来源/时间格式）

4. 召回和重排数量逻辑优化（固定 K 值困境）

            复杂问题信息缺失：如果该问题很综合，涉及的文档比较多，最后选择的参考资料却很少，那不足以回答问题
            
            简单问题噪声干扰：如果该问题很直接，涉及的文档仅一个，最后返回了很多无关资料会影响回答质量

5. 召回和重排的样本质量优化

            与知识库对接的时候，可能embedding/相关性指标不够合理，后续需要再详细研究（从优化prompt改写或者相关性指标入手）。

            尤其是英文：变个复数就没检索到。（Ablation Studies->Ablation Study）
