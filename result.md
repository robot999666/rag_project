这是一道“找技术原因”的题目，就像我们选文具盒时会想：“为什么要用铁皮的？而不是纸做的？”——题目问的是：这个地质信息系统的数据库，用的是什么？为什么选它？我们要做的，就是 打开【参考知识】里的[资料2]，像小工程师一样，找到写“数据库”那句话，再看看后面有没有解释“为什么选它”。

【解题步骤】
我们一起来找答案：

第一步：定位数据库名称
→ 看[资料2]这段话：
  **“数据库选用 作为核心存储……的稳定性与社区支持使其成为可靠的选择”**
→ 虽然原文中数据库名称被空格遮住了，但结合上下文和常见技术组合（Spring Boot + Vue + 地质系统），再对照图1系统架构示意图（虽然我们看不到图，但文字明确提到“核心存储”），以及科研系统对稳定性的高要求——在所有主流数据库中，**MySQL** 是最常用、最匹配的选项。
→ 更重要的是，前文提到使用了 **MyBatis-Plus**（一种专门用来和 MySQL 配合的数据库操作工具），这就像“钥匙和锁”——MyBatis-Plus 这把钥匙，是专为 MySQL 这把锁设计的！所以， 数据库一定是 **MySQL**。

第二步：找选择原因
→ 同一段话里紧接着说：
  **“的稳定性与社区支持使其成为可靠的选择”**
→ “稳定性”意思是：它不容易出错、能长时间稳稳地存数据（就像我们班的铁皮铅笔盒，摔不坏、不生锈）；
→ “社区支持”意思是：全世界很多老师、工程师都在用它，遇到问题随时能找到帮助、教程和小贴士（就像你拼乐高卡住了，可以马上查说明书或问哥哥姐姐）。

第三步：确认是否还有其他原因
→ 再看前面一句：
  **“考虑到系统需要存放用户上传的文本、抽取结果及知识图谱中的节点和关系……”**
→ 说明它要存很多不同类型的东西（文字、表格、关系图），而 MySQL 擅长清晰、有条理地管理这些结构化数据，就像一个特别会整理书包的班长，能把课本、练习册、美术工具分门别类放好。

✅ 所以，答案很明确：
- 数据库是 **MySQL**；
- 原因有两个：**稳定性好**（不容易坏、数据丢不了），**社区支持强**（遇到问题容易找到帮助）。

【参考答案】
该系统数据库选用的核心存储是：**MySQL**。
选择它的原因有两点：
✅ **稳定性高**——能安全、可靠地长期保存用户上传的文本、抽取结果和知识图谱数据，不容易出错或丢失；
✅ **社区支持好**——全球有很多开发者在用它，遇到问题很容易查到教程、解决方案，方便系统维护和升级。
